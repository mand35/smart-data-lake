Some Data Objects need a connection, e.g. JdbcTableDataObject, as they need to know how to connect to a database. Instead of defining the connection information for every data object, you can conveniently define it in one place and just use the reference in the data objects. The possible parameters depend on the connection type. Please note the section on usernames and password.

For a list of all available connections, please consult the API docs directly.

In the package overview, you can also see the parameters available to each type of connection and which parameters are optional.
